AIM
----------------------------------------------------------------------------------------------------------------------------
An intelligent chatbot built using **LangChain**, **Chainlit**, and **Gemini LLM** that answers user queries by first retrieving information from a **preprocessed PDF document**, and only then falling back to the **internet (DuckDuckGo)** if the document lacks relevant content.



## 🚀 Features
--------------------------------------------------------------------------------------------------------------------
- 📚 **PDF-First Answering:** Always checks the embedded PDF content first using FAISS and semantic search.
- 🌐 **Internet Fallback:** Automatically searches the web via DuckDuckGo only when the PDF lacks sufficient information.
- 💬 **Chat Interface with Chainlit:** Real-time chat UI built using Chainlit.
- 🧠 **Powered by Gemini LLM:** High-quality answers using Google's latest generative model.
- 🛠️ **RAG Pipeline Architecture:** Based on LangChain's Retrieval-Augmented Generation flow.



## 🧩 Tech Stack
-------------------------------------------------------------------------------------------------------------------

| Component              | Purpose                                                |
|------------------------|--------------------------------------------------------|
| **LangChain**          | Chains, tools, agents, and retrieval pipeline          |
| **Chainlit**           | Frontend chat interface                                |
| **FAISS**              | Vector search for semantic document retrieval          |
| **SentenceTransformers** | Embedding model: `all-mpnet-base-v2`                 |
| **DuckDuckGo Tool**    | Internet fallback for external queries                 |
| **Gemini LLM**         | Main language model via Google Generative AI           |

## 🧠 How It Works
---------------------------------------------------------------------------------------------------------

1. **Preprocessing**
   - Loads and chunks the PDF (using `RecursiveCharacterTextSplitter`)
   - Embeds chunks with SentenceTransformer
   - Stores them in FAISS vector DB

2. **Chat Interaction**
   - User query hits FAISS first using semantic similarity
   - If no good match, falls back to DuckDuckGo
   - Answers generated by Gemini LLM with sources if from PDF

3. **Fallback Logic**
   - Uses trigger phrases (like *“not in the document”*, *“i don’t know”*, etc.)
   - To decide when to fallback to internet search
